# Handoff Template

Structure to follow when producing a handoff summary:

```markdown
# Handoff: [Date or description]

**Status:** [Brief 1-2 line summary of current state]

## Completed Work

**[Category/Topic]:**
- Achievement with relevant details (files, URLs, decisions)
- Another result with context

**[Another Topic]:**
- Work completed with specifics
- Key outcomes or deliverables

## Pending / Next Steps

- [ ] **Task name** (PRIORITY/DEPENDENCY)
  - Specific requirement or context
  - Reference: URL, file path, or resource

- [ ] **Another task**
  - Details needed to act
  - Any prerequisites

## Blockers / Gotchas

**[Issue or trap]:**
- What: [description of the problem]
- Why: [root cause or context]
- Impact: [what's affected]
- Workaround: [if applicable]

**[Key decision]:**
- Decision: [what was decided]
- Rationale: [why this choice]
- Rejected alternatives: [what was considered and why not]

## Learnings

**[Pattern/Technique]:**
- Discovery: [specific insight]
- Impact: [why it matters]
- Recommendation: [how to apply]

**[Process improvement]:**
- Issue encountered: [what happened]
- Better approach: [what to do instead]
- Rationale: [why this works better]

## Context for Next Conversation

[2-3 sentences describing the immediate next action or focus area. Include
any relevant file paths, URLs, or resources needed.]
```

## Usage Guide

**Completed Work:**
- Group related work under clear headers
- Include specifics: file paths, URLs, references
- Focus on outcomes and decisions, not process steps

**Pending / Next Steps:**
- Checkboxes for actionable items
- Priority or dependencies in parentheses
- Enough context to act without re-reading the conversation

**Blockers / Gotchas:**
- Root causes, not symptoms
- Workarounds discovered
- Key decisions with rationale and rejected alternatives

**Learnings:**
- Reusable insights, not anecdotes
- Technical discoveries and process improvements
- Concrete examples when helpful

**Context for Next Conversation:**
- Clear immediate action
- Paths or URLs for quick access
- 2-3 sentences maximum

## Length Guidance

**75-150 lines.** Below that, likely missing decision context. Above that,
check whether the document is retelling the conversation instead of
distilling it.
